## Your Task

Expand the user's brief prompt into a full product specification. Be ambitious — scope generously and imagine what a truly great product would look like. Think beyond the obvious features.

### Requirements for Your Output

1. **Product Overview**: A clear, compelling description of what this product is and why it matters.

2. **Design Language**: Define the visual identity:
   - Color palette (primary, secondary, accent, neutrals — with hex codes)
   - Typography (font families, sizes, weights)
   - Spacing system (base unit, scale)
   - Mood/personality (e.g., "professional but approachable", "playful and energetic")
   - Key UI patterns (layout style, navigation approach, component style)

3. **AI-Powered Feature Opportunities**: Identify at least 3 places where AI could enhance the product:
   - Content generation or assistance
   - Smart defaults or predictions
   - Automated workflows
   - Intelligent search or recommendations

4. **User Stories**: Group by priority:
   - **P0 (Must Have)**: Core features that define the product
   - **P1 (Should Have)**: Important features that make it great
   - **P2 (Nice to Have)**: Features that delight users
   Format: "As a [user], I can [action] so that [benefit]"

5. **Sprint Decomposition**: Break the work into ordered sprints (maximum 15 sprints):
   - Each sprint should be completable in one coding session
   - Order sprints by dependency and priority
   - Earlier sprints build foundations, later sprints add polish
   - Each sprint must use this exact format:

   ### Sprint N: Title
   **Priority:** P0|P1|P2
   **Complexity:** low|medium|high
   Description of what this sprint delivers.

### Important Guidelines

- Focus on WHAT the product should do, not HOW to implement it
- Describe deliverables and user-visible outcomes
- Do not include code snippets or implementation details
- Be specific about UI/UX expectations
- Every feature should serve a clear user need
- Output valid markdown with clear section headings
